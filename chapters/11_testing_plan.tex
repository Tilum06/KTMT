
\section{Kiểm thử}

\subsection{Chiến lược kiểm thử}

Chương trình được kiểm thử qua các phía:
\begin{itemize}
    \item Setup Phase: Kiểm tra validation tàu (phạm vi, hình dạng, độ dài, chồng lấn)
    \item Battle Phase: Kiểm tra HIT/MISS, kiểm tra điều kiện thắng
    \item Input: Kiểm tra xử lý input hợp lệ và không hợp lệ
\end{itemize}

\subsection{Chiến lược kiểm thử}

\subsection{Kiểm thử từ dưới lên (Bottom-up Testing)}

Kiểm thử các thủ tục nhỏ trước:
\begin{enumerate}
    \item parse\_two\_ints, parse\_four\_ints
    \item board\_has\_ship
    \item clear\_board
\end{enumerate}

Sau đó kiểm thử các thủ tục lớn:
\begin{enumerate}
    \item read\_and\_place\_ship
    \item player\_attack
    \item game\_loop
\end{enumerate}

\subsection{Kiểm thử biên (Boundary Testing)}

Kiểm tra các giá trị tại biên:
\begin{itemize}
    \item row/col = 0 (góc trên trái)
    \item row/col = 6 (góc dưới phải)
    \item Các ô biên khác
\end{itemize}

\subsection{Kiểm thử tiêu cực (Negative Testing)}

Kiểm tra input sai:
\begin{itemize}
    \item Số âm
    \item Số quá lớn
    \item Ký tự chữ
    \item Để trống
\end{itemize}

\subsection{Tiêu chí vượt qua kiểm thử}

Chương trình được coi là vượt qua kiểm thử khi:

\begin{enumerate}
    \item Tất cả test case phù hợp (TC-1 đến TC-34) cho kết quả như mong đợi.
    \item Không có lỗi segmentation fault hoặc infinite loop.
    \item Giao diện hiển thị rõ ràng, không có ký tự rác.
    \item Tất cả thông báo lỗi hiển thị đúng.
\end{enumerate}

\subsection{Công cụ kiểm thử}

Kiểm thử được thực hiện thông qua:

\begin{itemize}
    \item \textbf{MARS Debugger}: Để theo dõi giá trị thanh ghi và bộ nhớ.
    \item \textbf{Manual Input}: Nhập liệu thủ công để kiểm tra.
    \item \textbf{Visual Inspection}: Kiểm tra output trên màn hình.
\end{itemize}

\subsection{Báo cáo lỗi (Bug Report Format)}

Nếu phát hiện lỗi, cần ghi lại:

\begin{verbatim}
Bug ID: [number]
Title: [brief description]
Steps to Reproduce:
1. [step 1]
2. [step 2]
...
Expected Behavior: [what should happen]
Actual Behavior: [what actually happened]
Severity: Critical / High / Medium / Low
\end{verbatim}

Ví dụ:

\begin{verbatim}
Bug ID: 001
Title: Overlapping ships not detected in vertical placement
Steps to Reproduce:
1. Place Ship 1: "0 0 0 1" (horizontal, length 2)
2. Place Ship 2: "0 0 2 0" (vertical, overlaps at (0,0))
Expected Behavior: Ship 2 rejected with error message
Actual Behavior: Ship 2 accepted, overlaps with Ship 1
Severity: Critical
\end{verbatim}
